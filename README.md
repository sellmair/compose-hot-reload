## Intro

This repository contains recent experiments for Hot Reloading Compose Applications.  
The intent is to upstream this repository into an official JetBrains product.

No guarantees apply.

## State

The project publishes experimental builds

### Add the 'firework' maven repository

(settings.gradle.kts)

```kotlin
pluginManagement {
    repositories {
        maven("https://packages.jetbrains.team/maven/p/firework/dev")
    }
}

dependencyResolutionManagement {
    repositories {
        maven("https://packages.jetbrains.team/maven/p/firework/dev")
    }
}

```

### Apply the Gradle plugin to your project

```kotlin
plugins {
    kotlin("multiplatform") version "2.1.0-firework.32" // <- Use special builds of Kotlin
    kotlin("plugin.compose") version "2.1.0-firework.32" // <- Use special builds of Kotlin/Compose Compiler
    id("org.jetbrains.compose")
    id("org.jetbrains.compose-hot-reload") version "1.0.0-dev.31.9" // <- add this additionally
}
```

### Enable 'OptimizeNonSkippingGroups' in your build.gradle.kts

```kotlin
composeCompiler {
    featureFlags.add(ComposeFeatureFlag.OptimizeNonSkippingGroups)
}
```

### Optional: Create a custom entry point to launch your hot application

```kotlin
// build.gradle.kts
tasks.register<ComposeHotRun>("runHot") {
    mainClass.set("my.app.MainKt")
}
```
